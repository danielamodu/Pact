Developer Guide

Universal Accounts & Session Keys

Learn how Pact combines EIP-7702 smart account upgrades with secure Trusted Execution Environments (TEEs) to execute non-custodial recurring pull payments.

Overview

Pact Protocol enables gas-efficient, secure, and fully decentralized automated payments by upgrading a user's standard Externally Owned Account (EOA) to a smart contract wallet under EIP-7702. By signing a session delegation scope, users authorize merchants to trigger periodic token pulls from their account within strict, cryptographic boundaries.

Key Architectures

• Trusted Execution Enclave: Houses secure wallet signing keys away from client exposure.
• EIP-7702 Delegation: Temporarily upgrades the user's account behavior to act as a Smart Account.
• Session Limits: Restricted permission limits block merchants from over-charging or draining wallets.

TEE Integration

Cryptographic signatures are generated using the Magic Labs Trusted Execution Environment (TEE). Client applications communicate with the backend TEE proxy to request signatures over data hashes or arbitrary strings.

typescript
import { getOrCreateWallet, signData } from "@/lib/express-proxy";

// 1. Retrieve the TEE wallet public address
const wallet = await getOrCreateWallet("ETH");
console.log("Public Address:", wallet.public_address);

// 2. Request TEE to sign transaction/authorization data
const sig = await signData(rawDataHash, "ETH");
console.log("Signature:", sig.r, sig.s, sig.v);

Session Keys

Rather than prompt users for interactive signatures on every transaction, Pact uses ephemeral session keys. An ephemeral EOA is generated in-browser and delegated specific pull permissions. The merchant uses this private key to execute scoped actions.

Formatting Delegation Statement

The delegation scope is formatted into a standardized human-readable statement to be cryptographically signed by the user's main wallet.

typescript
import { generateSessionKey, formatScopeStatement } from "@/lib/sessionKey";

const sessionKey = generateSessionKey();
const statement = formatScopeStatement({
  sessionKeyAddress: sessionKey.address,
  recipient: "0x_merchant_address",
  maxAmount: "49.99",
  token: "USDC",
  interval: 2592000, // 30 days
  expiry: 1768800000,
  planId: 1
});

Session Verification

Merchants verify the validity of delegation signatures off-chain to confirm that the session key is authorized by the true smart account owner.

typescript
import { verifySessionKeyDelegation } from "@/lib/sessionKey";

const isValid = verifySessionKeyDelegation(ownerAddress, delegation);
console.log("Session Authorized:", isValid);

Expiry Constraints

Delegations will automatically be marked invalid if the current block timestamp exceeds the delegation scope's expiry parameter.

Pact Registry Contract

All plans and subscription details are managed on-chain by the PactRegistry contract deployed at:

Contract Address (Base/Arbitrum)0xF2cDBAcdcE2D7961C2dE969E28F623ffac617CF5

Merchants deploy plans programmatically. When a user subscribes, a registration is registered under the registry mapping.

import { createPlanOnchain } from "@/lib/contracts";

const txHash = await createPlanOnchain(
  "base",
  "Professional Plan",
  usdcContractAddress,
  priceInSixDecimals,
  2592000, // 30 days
  merchantPayoutAddress
);

EIP-7702 Account Upgrades

Under EIP-7702, a user upgrades their standard EOA account to delegate control to the Smart Contract Executor (e.g. SessionKeyExecutor).

import { sign7702Authorization } from "@/lib/eip7702";

const auth = await sign7702Authorization({
  contractAddress: sessionKeyExecutorAddress,
  chainId: 8453, // Base Mainnet
  nonce: eoaNonce
});

Cross-Chain Pull Payments

Pact scans multiple networks (Arbitrum One and Base Mainnet) to find funds. If USDC or native ETH balances exist, relayers execute the pull payment via a cross-chain router.

import { checkAllBalances } from "@/lib/crossChainPull";

const balances = await checkAllBalances(userAddress);
console.log("USDC on Base:", balances.base.usdc);
console.log("USDC on Arbitrum:", balances.arbitrum.usdc);

Client Integration

The client app uses `expressProxy` helper functions to trigger secure enclave signing requests.

import { signMessage } from "@/lib/express-proxy";

const res = await signMessage(base64Message, "ETH");
console.log("Signature:", res.signature);

Relayer Service

The backend service monitors on-chain events and fires payment pulls on-time using the ephemeral session keys saved inside the databases.

Gas Optimization

Pact utilizes low-level execution proxies and minimizes on-chain storage to guarantee fee-free registrations and extremely cheap transactions.
